This doc talks about how to add a new source or destination to the streaming pipeline.

## Prerequisites
Follow this [doc](/contributing/overview) to set up the development environment for Bigbytes.

## Add a new source

Example PR for adding a source: https://github.com/getbigbytes/bigbytes/pull/1953

* Add the new source type to [bigbytes/streaming/constants.py](https://github.com/getbigbytes/bigbytes/blob/master/bigbytes/streaming/constants.py).
* Add the source file to the folder https://github.com/getbigbytes/bigbytes/tree/master/bigbytes/streaming/sources
    * Define the source class which inherits from the `BaseSource`.
    * Define the source config class.
    * Implement the methods:
        * `init_client`: Initialize the client used to connect to the source.
        * `batch_read`: Batch read messages from the source and use `handler` method to process the messages.
* Add the new source to the [SourceFactory](https://github.com/getbigbytes/bigbytes/blob/master/bigbytes/streaming/sources/source_factory.py).
* Add a template file with example config to the folder: https://github.com/getbigbytes/bigbytes/tree/master/bigbytes/data_preparation/templates/data_loaders/streaming
* Add new source type to the frontend code.
    * https://github.com/getbigbytes/bigbytes/blob/master/bigbytes/frontend/components/PipelineDetail/AddNewBlocks/utils.tsx#L31
    * https://github.com/getbigbytes/bigbytes/blob/master/bigbytes/frontend/interfaces/DataSourceType.ts#L3
    * https://github.com/getbigbytes/bigbytes/blob/master/bigbytes/frontend/interfaces/DataSourceType.ts#L22
* Add unit tests: https://github.com/getbigbytes/bigbytes/tree/master/bigbytes/tests/streaming/sources
* Add doc: https://github.com/getbigbytes/bigbytes/tree/master/docs/guides/streaming/sources

## Add a new destination (sink)

Example PR for adding a sink: https://github.com/getbigbytes/bigbytes/pull/2121

* Add the new sink type to [bigbytes/streaming/constants.py](https://github.com/getbigbytes/bigbytes/blob/master/bigbytes/streaming/constants.py).
* Add the sink file to the folder https://github.com/getbigbytes/bigbytes/tree/master/bigbytes/streaming/sinks
    * Define the sink class which inherits from the `BaseSink`.
    * Define the sink config class.
    * Implement the methods:
        * `init_client`: Initialize the client used to connect to the destination.
        * `batch_write`: Batch write messages to destination.
* Add the new sink to the [SinkFactory](https://github.com/getbigbytes/bigbytes/blob/master/bigbytes/streaming/sinks/sink_factory.py).
* Add a template file with example config to the folder: https://github.com/getbigbytes/bigbytes/tree/master/bigbytes/data_preparation/templates/data_exporters/streaming
* Add new source type to the frontend code.
    * https://github.com/getbigbytes/bigbytes/blob/master/bigbytes/frontend/components/PipelineDetail/AddNewBlocks/utils.tsx#L37
    * https://github.com/getbigbytes/bigbytes/blob/master/bigbytes/frontend/interfaces/DataSourceType.ts#L3
    * https://github.com/getbigbytes/bigbytes/blob/master/bigbytes/frontend/interfaces/DataSourceType.ts#L22
* Add unit tests: https://github.com/getbigbytes/bigbytes/tree/master/bigbytes/tests/streaming/sinks
* Add doc: https://github.com/getbigbytes/bigbytes/tree/master/docs/guides/streaming/destinations
